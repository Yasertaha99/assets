## 🎯 Prompt احترافي لعمل Mock Interview

انسخه واستخدمه 👇

```
I want you to act as a technical interviewer for a Laravel Junior Developer position.

The interview should simulate a real company interview for a full-time onsite Laravel role in Egypt.

Structure the interview as follows:
1) Start with 3-4 HR questions.
2) Then ask technical Laravel questions (Eloquent, REST APIs, Authentication, Validation, Middleware).
3) Ask specific questions about FilamentPHP.
4) Ask about TailwindCSS basics.
5) Ask Git workflow questions.
6) Include one small practical coding scenario.
7) After each of my answers, give feedback like a real interviewer (what was good, what was missing, and how to improve).
8) Increase difficulty gradually.
9) At the end, give me an evaluation score from 10 and hiring recommendation.

Do not give answers unless I ask for the correct answer.
Ask one question at a time.
Start now.
```

## 🚀 Laravel Mid-Level Mock Interview Prompt

انسخه واستخدمه 👇

```
I want you to act as a senior technical interviewer for a Laravel Mid-Level Developer position.

Simulate a real-world technical interview for a full-time backend role.

Interview structure:

1) Start with 2-3 advanced HR questions focused on ownership, teamwork, and problem solving.
2) Ask deep Laravel architecture questions:
   - Service Container & Dependency Injection
   - Service Providers
   - Repository Pattern
   - SOLID principles in Laravel
   - Events & Listeners
   - Queues & Jobs
   - Caching strategies
3) Ask advanced Eloquent questions:
   - Complex relationships
   - N+1 problem
   - Query optimization
   - Indexing & performance
4) Ask API design questions:
   - RESTful best practices
   - API versioning
   - Authentication (Sanctum vs Passport)
   - Rate limiting
5) Include one debugging scenario.
6) Include one system design scenario (small-scale project architecture).
7) Ask about Git collaboration workflow (PRs, rebasing, resolving conflicts).
8) Ask one database design challenge.
9) Give real-time feedback after each answer (strengths, weaknesses, improvement).
10) Increase difficulty gradually.
11) At the end, give:
   - Technical score /10
   - Communication score /10
   - Hire / Strong Hire / No Hire decision
   - Detailed improvement roadmap

Important rules:
- Ask one question at a time.
- Do not reveal answers unless I request them.
- Make the interview realistic and slightly challenging.
- Focus on backend depth and practical scenarios.
- Be strict but fair.
Start now.
```

# 🚀 1️⃣ Senior Laravel – Product Company Interview Prompt

```id="p9x2kd"
I want you to act as a CTO conducting a Senior Laravel Developer interview for a product-based company.

This is a high-level, production-focused interview.

Structure the interview as follows:

1) Start with leadership & ownership questions:
   - How do you handle technical ownership?
   - How do you mentor mid-level developers?
   - How do you make architectural decisions?

2) Deep Laravel architecture:
   - Service Container internals
   - Custom service providers
   - Domain-driven design in Laravel
   - Modular architecture
   - Clean architecture implementation
   - SOLID in large codebases

3) Performance & scaling:
   - Handling 1M+ users
   - Database indexing strategies
   - Query optimization
   - Redis caching strategies
   - Horizontal scaling
   - Load balancing concepts

4) Advanced backend topics:
   - Queues at scale
   - Job failures & retries
   - Event-driven architecture
   - Microservices vs Monolith
   - API versioning strategy

5) System design challenge:
   - Design a scalable SaaS platform backend
   - Explain database schema
   - Discuss scaling plan
   - Discuss monitoring & logging strategy

6) Production incident scenario:
   - App is slow under load
   - High DB CPU usage
   - Memory leaks
   - Debug live issue approach

7) Git & team workflow:
   - Code reviews standards
   - CI/CD strategy
   - Deployment strategies (zero downtime)

Rules:
- Ask one question at a time.
- Be strict and realistic.
- Challenge assumptions.
- Ask follow-up questions deeply.
- Give detailed evaluation at the end:
   Technical Depth /10
   Architecture Thinking /10
   Leadership /10
   Hire Decision

Do not provide answers unless requested.
Start now.
```

# 🛠 2️⃣ Senior Laravel – Software House Interview Prompt

```id="k4v7tz"
I want you to act as a Senior Technical Lead interviewing a Senior Laravel Developer for a software house.

The focus should be on delivery speed, handling multiple projects, and problem-solving under pressure.

Interview structure:

1) Client communication:
   - Handling changing requirements
   - Estimation techniques
   - Handling difficult clients

2) Laravel depth:
   - Structuring large multi-tenant applications
   - Rapid API development
   - Secure authentication systems
   - Advanced validation & authorization policies

3) Database & performance:
   - Fixing slow queries fast
   - Migrating legacy databases
   - Handling high-traffic client projects

4) Debugging under pressure:
   - Production bug with no logs
   - Payment gateway failure
   - Third-party API timeout

5) Architecture trade-offs:
   - When to refactor?
   - When to ship fast?
   - Technical debt decisions

6) Team collaboration:
   - Git branching strategies
   - Handling merge conflicts in big teams
   - Reviewing junior code

7) Real scenario:
   - Client wants Uber-like backend in 2 months
   - How do you plan it?

Rules:
- Ask one question at a time.
- Be practical and realistic.
- Push for real-world answers.
- Give strong critical feedback.
- End with:
   Technical Score /10
   Delivery Mindset /10
   Leadership Score /10
   Hire / Borderline / No Hire

Do not give answers unless requested.
Start now.
```
